    J -- "否" --> L["完成"]
    K --> L
关键时序要点
  • ① 中：timerfd 在 TimerQueue 构造时就被挂上 epoll，但没 timerfd_settime 之前永远不可读，挂上无害。
  • ② 中：arm_idle_timer 已经在 subLoop 线程里执行（因为 connection_cb_ 是被 mainLoop 用 run_in_loop 派进来的），所以后续 run_after → add_timer_in_loop → timerfd_settime 全部同步完成，零跨线程开销。
  • ③ 中：handle_read 把到期项 extract 出 timers_ 时，必须同步 id_to_iter_.erase(id)，否则后续 cancel 会拿到已失效的 iterator。
  • ④ 中：cancel 整体复杂度 O(log N)（平均 O(1) 查 map + O(log N) erase set），相比之前 O(N) 线性扫描，连接数大时差异显著。
